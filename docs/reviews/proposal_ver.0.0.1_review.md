# 2. P0必須修正

## P0-001：P0要件定義工程を企画書に追加する

### 問題

現行企画書では、企画から実装マイルストーンへ進む流れはありますが、**P0要件定義工程**が独立していません。

### 修正内容

「P0：初期実装スコープ確定」を、以下のように明記します。

```text
P0：初期実装スコープ確定は、実装前のMVP/P0要件定義工程とする。
この工程の成果物は docs/research/research-memo-builder-p0-requirements.md とする。
```

### 理由

実装前に要件を固定しないと、検索API接続、出力ファイル、Markdown生成、ChatGPTプロンプト生成の範囲が膨らみやすいためです。

---

## P0-002：MVP / P0 / P1 の境界を整理する

### 問題

現行企画書では、MVPでやることに以下が含まれています。

- JSON出力
- 重複排除
- ChatGPT分析プロンプト出力

一方で、初期実装の優先順位では、これらがP1に分類されています。

### 修正案

Active化時点では、以下のように整理するのがよいです。

| 区分 | 内容                                                                                                                    |
| ---- | ----------------------------------------------------------------------------------------------------------------------- |
| P0   | APIキー読み込み、入力YAML読み込み、検索クエリ生成、Brave API接続、検索結果正規化、CSV出力、Markdownリサーチメモ最小出力 |
| P1   | JSON出力、重複排除、実行レポート、ChatGPT分析プロンプト出力                                                             |
| P2   | キャッシュ、RSS連携、手動URL投入                                                                                        |
| P3   | LLM API連携、Mnemosyne連携                                                                                              |

### 理由

まずは「検索して、リサーチメモの下書きが出る」ことをP0完了条件にした方が、初期実装が小さくなります。

## P0-003：MVPスコープの表現を修正する

### 問題

「MVPでやること」と「P0でやること」が混ざっています。

### 修正内容

企画書の「MVPスコープ」を次の2段構成に変更します。

```markdown
## MVP全体スコープ

MVP全体では、Brave Search APIから検索結果を取得し、CSV、JSON、Markdownリサーチメモ、ChatGPT分析プロンプトを出力できる状態を目指す。

## P0初期実装スコープ

P0では、まず以下に限定する。

- `.env` による `BRAVE_API_KEY` 読み込み

- Markdownリサーチメモの最小出力
```

### 理由

MVP全体の完成像と、最初に作るP0実装を分けるためです。

# 3. P1推奨修正

## P1-001：`extra_snippets` の扱いを明記する

### 現状

`extraSnippets: true` が入力YAML例に入っています。

### 修正案

```text
P0では extra_snippets は設定可能項目とする。
初期値は true とするが、取得できない場合でも処理を継続する。
```

### 理由

切り口分析や無料部分の約束推定に有用ですが、必須依存にするとAPIレスポンス変更時に壊れやすいためです。

---

## P1-002：JSON出力の優先度を再整理する

### 判断案

JSON出力はP0に入れてもよいですが、P0を小さくするならP1でよいです。

おすすめは以下です。

| 出力                  | 優先度 | 理由                                                     |
| --------------------- | ------ | -------------------------------------------------------- |
| CSV                   | P0     | 検索結果確認に必須                                       |
| Markdownリサーチメモ  | P0     | 目的そのものに直結                                       |
| JSON                  | P1     | デバッグ・将来連携に有用だが初期必須ではない             |
| run-report            | P1     | 品質向上には有用だが初期必須ではない                     |
| ChatGPT分析プロンプト | P1     | 手動分析補助として有用だが、P0ではMarkdownメモで代替可能 |

---

## P1-003：重複排除の優先度を再整理する

### 現状

マイルストーンM3では重複URL除外が完了条件に含まれていますが、優先順位表では重複排除がP1です。

### 修正案

どちらかに統一します。

おすすめ：

```text
P0では同一URLの単純重複除外のみ行う。
タイトル類似などの高度な重複判定はP1以降にする。
```

### 理由

同一URL重複はCSVやMarkdownの品質に直結するため、最小実装に入れた方がよいです。
ただし、タイトル類似や正規URL化まで入れると重くなるため、P1以降でよいです。

---

## P1-004：企画書と要件定義の責務分離を明記する

### 修正案

企画書には以下を追加します。

```text
本企画書は、Research Memo Builderの目的、背景、採用方針、MVP全体像、リスク、マイルストーンを定義する。

個別のP0実装要件は、docs/research/research-memo-builder-p0-requirements.md に切り出す。
```

### 理由

企画書が要件定義・基本設計・実装仕様まで抱え込むと、Active化後に保守しづらくなります。
